# Handoff: V2.1 Sales Workspace memory merge and auto-write bugfix

Date: 2026-04-29

## Changed

- `backend/sales_workspace/chat_first.py`
  - ContextPack now includes full current product profile / lead direction and the last 12 same-thread messages with longer excerpts.
- `backend/runtime/sales_workspace_chat_turn_llm.py`
  - Prompt now treats existing formal cards as the memory baseline.
  - Added merge guard for product profile and lead direction patch payloads.
  - Placeholder values such as "待确认", "未知", "未提供", and "用户未说明" no longer overwrite confirmed fields.
  - `lead_direction_update` drops LLM-generated product-profile operations so customer-finding turns cannot degrade product understanding.
- `backend/api/sales_workspace.py`
  - Chat-first LLM turns now materialize and apply valid patch drafts immediately.
  - Draft reviews are kept as audit records with `status=applied` and an apply result.
  - Assistant copy now says the information was沉淀到工作区 for auto-applied updates.
- `backend/tests/test_sales_workspace_chat_first_llm_runtime.py`
  - Updated expectations from preview-only to auto-applied behavior.
  - Added regression tests for product-profile non-degradation and incremental product merge.
- `app/src/main/java/com/openclaw/android/nativeentry/ui/OpenClawApp.kt`
  - Refreshes workspace snapshot after a successful Sales Agent turn.
- `app/src/main/java/com/openclaw/android/nativeentry/ui/workspace/SalesWorkspaceScreen.kt`
  - Shows applied chat-first updates as "已沉淀到工作区" and hides main-flow review/write buttons for applied reviews.

## Known limits

- Auto-write applies only to V2.1 product profile and lead direction updates generated by the chat-first LLM runtime.
- `workspace_question` remains read-only and does not mutate workspace memory.
- No V2.2 search/contact/CRM/company-list capability was added.
- Existing draft-review API routes remain for legacy/debug flows, but the main chat-first path now auto-applies valid updates.
